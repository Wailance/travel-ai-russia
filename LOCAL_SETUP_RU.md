# Локальный запуск Travel AI Russia

Пошаговая инструкция после распаковки архива.

## 1) Что нужно заранее

- Node.js 20+ ([nodejs.org](https://nodejs.org))
- Интернет (GigaChat API, карта OpenStreetMap, геокодинг)
- Ключ GigaChat из [developers.sber.ru/studio](https://developers.sber.ru/studio)

Проверка:

```bash
node -v
npm -v
```

## 2) Распаковка

Распакуйте архив в любую папку, например:

`Downloads/travel-ai-russia`

В терминале перейдите в папку проекта:

```bash
cd путь/к/travel-ai-russia
```

## 3) Установка зависимостей

```bash
npm install
```

## 4) Файл `.env`

В архиве уже есть готовый `.env` с ключом GigaChat — можно сразу запускать.

Если файла нет или нужен новый ключ:

```bash
cp .env.example .env
```

Откройте `.env` и укажите **ключ авторизации** GigaChat (рекомендуется):

```env
GIGACHAT_AUTH_KEY=ваш_ключ_авторизации_из_кабинета
GIGACHAT_TOKEN=
GIGACHAT_USE_OAUTH=true
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_DEMO_MODE=false
GIGACHAT_INSECURE_TLS=true
PORT=3000
```

- **Ключ авторизации** — строка Authorization key из кабинета (не путать с краткоживущим access token).
- Для физлиц обычно `GIGACHAT_SCOPE=GIGACHAT_API_PERS`, для B2B — `GIGACHAT_API_B2B`.
- `GIGACHAT_DEMO_MODE=true` — маршрут-заглушка без GigaChat (для проверки интерфейса).
- После изменения `.env` перезапустите сервер.

## 5) Фронтенд: `public/config.js`

Для локального запуска (сайт и API на одном компьютере) оставьте:

```js
window.TRAVEL_API_BASE = "";
```

Пустая строка — запросы идут на тот же сервер (`/api/plan`).

Для отдельного backend (Render/GitHub Pages):

```js
window.TRAVEL_API_BASE = "https://ваш-backend.onrender.com";
```

## 6) Запуск

```bash
npm start
```

или

```bash
npm run dev
```

Откройте в браузере: [http://localhost:3000](http://localhost:3000)

**Важно:** не открывайте `index.html` двойным щелчком — только через адрес выше.

## 7) Возможности приложения

В форме задаются:

| Параметр | Описание |
|----------|----------|
| Срок (дней) | 1–30 |
| Город старта / окончания | Города России |
| Количество городов | 2–10, не больше числа дней (включая старт и финиш) |
| Бюджет | В рублях |
| Проживание / своё авто | Чекбоксы |
| **Цель поездки** | До **3** целей из списка (см. ниже) |

**Цели поездки:** сбалансированный маршрут (отдельно, не сочетается с другими), достопримечательности, культурное просвещение, гастрономия, православие, природа, семья, романтика, активный отдых, шопинг, фото, история и патриотика, ночная жизнь.

**Результат:** маршрут, расписание по дням, логистика, бюджет, графики, интерактивная карта (Leaflet + OpenStreetMap).

## 8) Если не работает

| Симптом | Что сделать |
|---------|-------------|
| «Failed to fetch» | Запустите `npm start`, откройте http://localhost:3000, в `config.js` — пустой `TRAVEL_API_BASE` |
| OAuth 401, code 6 | Новый **ключ авторизации** в `GIGACHAT_AUTH_KEY`, проверьте `GIGACHAT_SCOPE` |
| Карта пустая | Обновите страницу (Cmd+Shift+R), проверьте интернет |
| Маршрут долго строится | 15–40 с — нормально для GigaChat |
| Низкий бюджет | Увеличьте бюджет или сократите дни/города |
